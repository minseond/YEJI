# 화투 48장 카드 레퍼런스 (ai/develop 기준)

> 이미지 매핑 및 LLM 프롬프트 참조용 문서
> 소스: `ai/src/yeji_ai/models/enums/hwatu.py`

---

## 같은 월 피 2장, 구분하나?

실제 화투에서 같은 월 피 2장은 그림이 미묘하게 다르다 (예: 매화 위치, 꽃잎 수 등).
하지만 **화투점(우리 서비스) 에서는 구분하지 않는다.**

| 항목 | 설명 |
|------|------|
| `name_ko` | 동일 (예: code 6, 7 모두 `"2월 피"`) |
| LLM 전달 정보 | `card_code`(숫자), `card_type`("피"), `card_month`(월) → 둘 다 동일 |
| `fortune_meaning` | 약간 다르지만, 폴백(LLM 실패 시) 더미 텍스트 전용. LLM 프롬프트에 포함되지 않음 |
| 점술 결과 | **차이 없음** |

**결론:**
- 프론트에서 이미지를 다르게 넣어도, 같은 이미지를 공유해도 점술 결과는 동일
- 시각적 다양성을 위해 다른 이미지를 쓰는 건 자유

---

## 기본 규칙

```
총 48장 = 12월 × 4장
card_code: 0 ~ 47 (정수)

월 계산:      card_code // 4 + 1
월 내 순서:   card_code % 4  (0~3)
```

---

## 이미지 필요 수량

| 구분 | 장수 | 비고 |
|------|------|------|
| 고유 이미지 카드 | **33장** | 광 5, 열끗 12, 띠 10, 피(월당 1종) 6 |
| 동일 이미지 피 | 15장 | 같은 월 피 2장은 같은 그림 → 이미지 공유 |
| 전체 카드 | 48장 | |

같은 월의 **피 2장**은 실제 화투에서도 동일한 그림이며,
LLM에도 `card_code`(숫자)와 `card_type`("피")만 전달되므로 **이미지 1개로 공유 가능**.

## API 요청 예시

```json
{
  "category": "HWATU",
  "question": "오늘 금전운이 궁금해요",
  "cards": [
    {"position": 1, "cardCode": 0,  "isReversed": false},
    {"position": 2, "cardCode": 28, "isReversed": false},
    {"position": 3, "cardCode": 10, "isReversed": false},
    {"position": 4, "cardCode": 20, "isReversed": false}
  ]
}
```

- `cardCode`는 0~47 정수, 중복 불가
- `isReversed`는 항상 `false` (화투는 역방향 없음)
- `position`은 1~4, 모두 포함 필수

## 프론트엔드 이미지 매핑 코드 예시

```typescript
// card_code → 이미지 파일 매핑
function getHwatuImage(cardCode: number): string {
  // 같은 월 두 번째 피는 첫 번째 피 이미지 공유
  const PI_SHARED: Record<number, number> = {
    7: 6, 15: 14, 19: 18, 23: 22, 27: 26,
    31: 30, 35: 34, 39: 38, 43: 42,
  };

  const imageCode = PI_SHARED[cardCode] ?? cardCode;
  return `/images/hwatu/hwatu_${String(imageCode).padStart(2, '0')}.png`;
}
```
