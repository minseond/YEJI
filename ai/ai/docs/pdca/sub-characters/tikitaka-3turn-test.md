## 1. 검증 개요

### 1.1 목표
- 소이설(SOISEOL) → 서브캐릭터(4종) → 스텔라(STELLA) 순서로 3턴 연속 대화 시뮬레이션
- 각 캐릭터 간 자연스러운 대화 전환 확인
- 컨텍스트(이전 대화 내용) 유지 여부 검증
- 캐릭터 말투 일관성 확인

### 1.2 테스트 API 엔드포인트

| 엔드포인트 | 설명 | 지원 캐릭터 |
|------------|------|-------------|
| `/v1/fortune/chat` | 메인 티키타카 채팅 | SOISEOL, STELLA |
| `/v1/fortune/chat/test-character` | 서브 캐릭터 테스트 | CHEONGWOON, HWARIN, KYLE, ELARIA |

| STELLA | GREETING | "안녕하세요! 저는 스텔라예요. 별자리와 행성 배치로 당신의 운명을 분석해드릴게요." | **해요체 O** |

## 3. 시나리오 2: 서브 캐릭터 자기소개

### 3.1 청운 (CHEONGWOON)

**요청:**
```json
{
  "character": "CHEONGWOON",
  "message": "자기 소개를 해주세요."
}
```

**응답:**
> "이 늙은이는 동방의 신선이며, 소이설(逍隠)의 스승이오. 천지의 이치와 사주, 철학을 깊이 연구하여 지혜로운 조언을 내리는 존재이니..."

**패턴 분석:**

- 응답 후반부에 외국어(태국어) 및 반복 패턴 발생
- 하오체 어미 감지 횟수 낮음 (1회)

**평가:**
- 말투 일관성: 6/10 (초반 하오체 유지, 후반 붕괴)
- 캐릭터 정체성: 7/10 ("늙은이", "신선" 등 키워드 O)

### 5.2 발견된 주요 문제

| 문제 유형 | 심각도 | 영향 캐릭터 | 설명 |
|-----------|--------|-------------|------|
| LLM 반복 출력 | **HIGH** | 전체 | 동일 텍스트 반복 또는 비정상 문자 출력 |
| 외국어 혼입 | **HIGH** | 서브 캐릭터 | 태국어, 아랍어 등 비정상 문자 출현 |

| 응답 길이 과다 | LOW | 전체 | max_tokens 제한 도달 |

### 5.3 권장 개선 사항

1. **LLM 반복 방지**
   - `presence_penalty` 값 조정 (현재 1.5 → 2.0 검토)
   - `stop_tokens` 추가 설정

2. **응답 후처리 강화**
   - 비정상 문자 필터링 로직 추가
   - 외국어 감지 및 제거

3. **말투 일관성 개선**

   - 화린: 합니다체 금지 규칙 강화
   - 카일: 반말 비중 증가를 위한 Few-shot 보강

4. **컨텍스트 전달**
