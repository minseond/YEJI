## 문제 원인 분석

### 1. LLM 응답 노이즈 (P2)
- **현상**: Thai, Russian, Hebrew, Arabic 등 외래 문자 출현
- **원인 추정**:
  - vLLM 토크나이저 다국어 학습 데이터 영향
  - Qwen3 모델의 다국어 특성
- **해결 방안**: postprocessor에서 비한글 문자 필터링
- **우선순위**: P2 (별도 태스크 #180)

### 2. 엘라리아 합니다체 혼재 (P1)
- **현상**: Few-shot 강화에도 합니다체 지속 사용
- **원인 분석**:
  1. "운세", "공식" 맥락에서 격식체 선호 경향
  2. 30개 예시가 충분하지 않음
  3. LLM이 프롬프트 후반부 지시를 더 잘 따름
- **시도한 해결책**:
  - 1차: Few-shot 30개 예시 추가
  - 2차: 합니다체 "최소화" 규칙
  - 3차: 합니다체 "절대 금지" + 변환 예시 강화
- **다음 시도**:
  - 프롬프트 구조 변경 (규칙을 마지막에 반복)
  - 또는 postprocessor에서 합니다체→해요체 자동 변환

### 3. 카일 반말 패턴 감지 부족 (P2)
- **현상**: regex가 "~어", "~다" 종결어미 미감지
- **원인**: 테스트 코드 regex 패턴 불완전
- **해결 방안**: regex 패턴 확장
- **우선순위**: P2 (테스트 코드 개선)
