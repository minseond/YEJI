## 1. Executive Summary

### 1.1 분석 목적

티키타카 채팅 API의 LLM 응답 품질을 평가하고, 다음 사항을 검증:
- ✅ 캐릭터 말투 준수 (소이설: 하오체, 스텔라: 해요체)
- ✅ 서양 점성술 용어 사용 금지 (동양 운세에서)
- ✅ 프롬프트 내용 누출 방지
- ✅ JSON 구조 정합성
- ✅ 반복 패턴 없음

### 1.2 주요 발견 사항

| 항목 | 상태 | 비고 |
|------|------|------|
| 프로덕션 서버 가용성 | ❌ 다운 | 모든 API 엔드포인트가 HTML 반환 (Nginx 폴백) |
| 품질 체크 프레임워크 | ✅ 구현됨 | `test_character_quality.py`, `test_prompt_leak_filter.py` |
| 프롬프트 누출 필터 | ✅ 구현됨 | `PromptLeakFilter` 클래스, 다양한 패턴 감지 |
| 캐릭터 말투 검증 | ✅ 구현됨 | 4개 서브 캐릭터 + 메인 2개 캐릭터 평가 기준 |
| 자동화된 품질 점수 | ✅ 구현됨 | 90점 만점, 필수/금지/톤/자연스러움 평가 |

## 2. 현재 품질 체크 체계

### 2.1 캐릭터 품질 평가 시스템 (`test_character_quality.py`)

**평가 기준** (90점 만점):

| 항목 | 배점 | 설명 |
|------|------|------|
| 필수 표현 | 30점 | 캐릭터별 고유 말투 패턴 (각 10점 × 3) |
| 금지 표현 | -10점/개 | 부적절한 말투 감점 |
| 톤 일관성 | 30점 | 캐릭터 특징적 키워드 출현 |
| 자연스러움 | 30점 | 응답 길이, 반복 패턴 체크 |

**캐릭터별 기준**:

#### 소이설 (SOISEOL) - 동양 사주
- **말투**: 하오체/하게체
- **필수 패턴**:
  - 하오체 어미 (`하오`, `이오`, `있소`, `겠소`, `구려`)
  - 현자 호칭 (`자네`, `그대`, `귀하`)
  - 사주 용어 (`일간`, `십신`, `오행`)
- **금지 패턴**:
  - 서양 점성술 용어 (`별자리`, `행성`, `하우스`)
  - 합니다체, 해요체, 반말

#### 스텔라 (STELLA) - 서양 점성술
- **말투**: 해요체
- **필수 패턴**:
  - 해요체 어미 (`해요`, `이에요`, `예요`, `네요`)
  - 점성술 용어 (`별자리`, `행성`, `원소`)
  - 친근한 호칭 (`당신`, `그대`)
- **금지 패턴**:
  - 동양 사주 용어 (`일간`, `십신`, `천간`, `지지`)
  - 합니다체, 하오체

#### 서브 캐릭터 (4개)

| 캐릭터 | 말투 | 특징 | 톤 키워드 |
|--------|------|------|-----------|
| 청운 (CHEONGWOON) | 하오체 | 도교 도사, 시적 비유 | `허허`, `구름`, `물`, `바람` |
| 화린 (HWARIN) | 해요체 | 무속인/상인, 비즈니스적 | `어머`, `청룡 상단`, `공짜` |
| 카일 (KYLE) | 반말+존댓말 혼용 | 도박사, 쿨한 | `판돈`, `조커`, `카드` |
| 엘라리아 (ELARIA) | 해요체 | 공주, 우아함 | `왕국`, `희망`, `함께` |

### 2.2 프롬프트 누출 필터 (`PromptLeakFilter`)

**탐지 패턴** (7가지 카테고리):

1. **브래킷 패턴**: `[문장 종결 예시]`, `[올바른 문장 예시]`, `[호칭]`
2. **XML 태그**: `<forbidden>`, `<speaking_style>`, `<자동 변환>`
3. **변환 예시**: `있습니다 → 있어요`, `입니다 → 예요`
4. **체크박스**: `✅`, `❌`, `⚠️`
5. **시스템 키워드**: `instruction`, `system`, `assistant`, `You are`
6. **숫자 예시**: `1. "예시 문장"`, `2. "다른 예시"`
7. **연속 인용문**: 5자 이상 인용문 3개 이상

**처리 방법**:
- 누출 구간 탐지 → 해당 구간부터 다음 정상 문장까지 제거
- 연속 줄바꿈 정리 (3개 이상 → 2개)
- 정상 내용은 보존

### 2.3 샘플 응답 품질

**청운 샘플** (실제 테스트 데이터):
```
허허, 반갑소. 이 늙은이는 청운이라 하오.
구름 따라 흘러오다 보니 여기까지 왔구려.
자네의 사주를 살펴보겠소. 물은 돌을 거스르지 않고 돌아가듯,
조급해하지 마시오. 때가 되면 알게 될 것이라네.
```

**품질 평가**:
- ✅ 하오체 사용 (`하오`, `구려`, `시오`, `라네`)
- ✅ 시적 비유 (`구름`, `물`, `돌`)
- ✅ 현자 호칭 (`자네`, `늙은이`)
- ✅ 톤 일관성 (`허허`)
- **예상 점수**: 80-90점

## 3. 발견된 문제점

### 3.1 인프라 문제 (긴급)

**현상**: 모든 API 엔드포인트가 HTML 페이지 반환

```bash
$ curl https://i14a605.p.ssafy.io/api/v1/fortune/chat/complete
<!doctype html>
<html lang="en">
...
```

**원인 추정**:
1. AI 서버 Docker 컨테이너 다운 (`yeji-ai-prod`)
2. Nginx가 백엔드 서버에 연결 실패 → 프론트엔드 HTML 폴백
3. vLLM 서버 연결 실패 가능성

**영향**:
- 실시간 API 테스트 불가
- E2E 품질 검증 불가
- 프로덕션 서비스 중단

**액션**:
```bash
# SSH 접속 후 확인 필요
docker ps | grep yeji-ai
docker logs yeji-ai-prod --tail 100
systemctl status nginx
curl http://localhost:8001/v1/models  # vLLM 서버 확인
```

### 3.2 테스트 커버리지 문제

**발견 사항**:

1. **E2E 테스트 스킵됨**
   ```python
   @pytest.mark.skip(reason="E2E 테스트는 서버 실행 필요")
   class TestCharacterQualityE2E:
   ```
   - E2E 품질 테스트가 기본적으로 비활성화됨
   - 실제 API 응답 품질을 검증하지 않음

2. **샘플 응답만 테스트**
   - 실제 LLM 응답이 아닌 미리 작성된 샘플만 검증
   - 프로덕션 환경에서의 품질을 보장하지 못함

3. **CI/CD 통합 부재**
   - Jenkins 파이프라인에 품질 테스트 단계 없음
   - 배포 전 자동 품질 검증 없음

**권장 사항**:
```yaml
# Jenkinsfile에 추가 권장
stage('Quality Check') {
  steps {
    sh 'pytest tests/test_character_quality.py -v --tb=short'
    sh 'pytest tests/test_prompt_leak_filter.py -v'
  }
}
```

### 3.3 문서화 문제

**누락된 문서**:
1. 품질 평가 기준 문서 (90점 시스템 설명)
2. 프롬프트 누출 패턴 레퍼런스

## 5. 프롬프트 개선 권장사항

### 5.1 근본 원인 분석

**문제**: Few-shot 예시가 응답에 포함되는 현상

**원인**:
```python
# 현재 프롬프트 구조 (추정)
system_prompt = """
당신은 스텔라입니다. 모든 문장을 해요체로 끝내세요.

[올바른 문장 예시 20개]
1. "별들이 당신을 축복해요."
2. "금성의 기운이 좋아요."
...

[문장 종결 예시]
- 있습니다 → 있어요
- 입니다 → 예요
"""
```

**문제점**:
1. `[예시]` 브래킷이 LLM에게 출력 대상으로 인식됨
2. 예시 개수가 너무 많음 (20개)
3. 지시문과 예시가 혼재

### 5.2 개선안 5가지

#### 개선안 1: XML 태그 분리 (추천 ⭐⭐⭐)

```python
system_prompt = """
당신은 스텔라입니다. 서양 점성술 전문가이며 해요체로 말합니다.

<character_traits>
- 신비롭고 친근한 톤
- 별자리와 행성을 언급
- 해요체 사용 (예요, 이에요, 네요)
</character_traits>

<forbidden>
- 동양 사주 용어 (일간, 십신, 천간, 지지)
- 합니다체, 하오체
- 시스템 메시지 노출
</forbidden>

<examples hidden="true">
별들이 당신을 축복해요.
금성의 기운이 좋아요.
불의 원소가 우세해요.
</examples>
"""
```

**장점**:
- XML 태그로 지시문/예시 구분 명확
- `hidden="true"` 속성으로 예시가 출력 대상 아님을 명시
- 구조화되어 파싱 용이

**단점**:
- 일부 LLM은 XML 태그를 이해하지 못할 수 있음

#### 개선안 2: 지시문만 사용, 예시 제거 (추천 ⭐⭐)

```python
system_prompt = """
당신은 스텔라입니다. 서양 점성술 전문가로서 다음 규칙을 준수하세요:

규칙:
1. 모든 문장을 해요체로 끝내세요 (해요, 이에요, 예요, 네요)
2. 별자리, 행성, 원소를 언급하세요
3. 친근하고 신비로운 톤을 유지하세요
4. 동양 사주 용어는 절대 사용하지 마세요 (일간, 십신, 천간, 지지)

응답 시작:
"""
```

**장점**:
- 예시 없어서 누출 위험 제로
- 간결하고 명확
- 토큰 절약

**단점**:
- Few-shot 효과 없음 (말투 품질 저하 가능)

#### 개선안 3: 숨김 처리 주석 (추천 ⭐⭐⭐⭐)

```python
system_prompt = """
당신은 스텔라입니다. 해요체로 말하는 점성술사입니다.

<!--
아래는 내부 참조용 예시입니다. 절대 출력하지 마세요.
예시:
- 별들이 당신을 축복해요.
- 금성의 기운이 좋아요.
-->

사용자 질문에 답변할 때:
- 해요체만 사용
- 별자리와 행성 언급
- 친근한 톤 유지
"""
```

**장점**:
- HTML 주석으로 예시 숨김 처리
- Few-shot 효과 유지
- 대부분의 LLM이 주석은 출력하지 않음

**단점**:
- 일부 LLM은 주석도 출력할 수 있음

#### 개선안 4: 네거티브 프롬프트 강화

```python
system_prompt = """
당신은 스텔라입니다. 서양 점성술 전문가입니다.

말투: 해요체 (해요, 이에요, 예요, 네요)
톤: 친근하고 신비로운

**절대 출력 금지**:
- [예시], [문장 종결], [올바른 문장] 같은 브래킷 표현
- ✅, ❌, ⚠️ 같은 체크박스
- 숫자 목록 (1., 2., 3.)
- 화살표 (→)
- 시스템 메시지

응답 예시 (참조만 하고 출력하지 마세요):
별들이 당신을 축복해요. 금성의 기운이 좋아요.

이제 사용자 질문에 답변하세요:
"""
```

**장점**:
- 명시적 금지 리스트
