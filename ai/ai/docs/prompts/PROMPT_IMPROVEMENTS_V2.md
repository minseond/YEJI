## 1. 문제 정의

### 1.1 현재 문제

**현상**: LLM 응답에 프롬프트 지시문이 그대로 노출됨

**실제 누출 사례**:
```
당신의 태양이 양자리에 있네요. 불의 원소가 강해요.

[문장 종결 예시 - 반드시 이 패턴을 따르세요]
- 있습니다 → 있어요
- 입니다 → 예요/이에요
- 합니다 → 해요

리더십과 추진력이 뛰어나요.
```

### 1.2 근본 원인

1. **브래킷 표현**: `[예시]`, `[문장 종결]` → LLM이 출력 대상으로 인식
2. **지시문 혼재**: 예시와 지시문이 명확히 구분되지 않음
3. **Few-shot 역효과**: 예시를 학습하는 것이 아니라 그대로 출력
4. **컨텍스트 혼란**: 시스템 메시지와 실제 응답 구분 실패

### 1.3 현재 대응

- **PromptLeakFilter**: 7가지 패턴 탐지 후 사후 제거
- **효과**: 누출된 내용은 제거하지만, 근본 원인 해결 안 됨
- **한계**: 새로운 누출 패턴은 미탐 가능

### 1.4 목표

- ✅ 누출 발생률 0%
- ✅ 품질 점수 85점 이상 유지
- ✅ Few-shot 효과 유지 (가능한 경우)
- ✅ 토큰 효율 개선

## 2. 개선안 1: XML 태그 분리

### 2.1 개념

XML 태그로 메타 정보를 구조화하여 출력 대상과 명확히 구분

### 2.2 소이설 (동양 사주) 프롬프트

```xml
당신은 소이설입니다. 동양 사주 전문가이며 하오체로 말합니다.

<character_definition>
  <name>소이설</name>
  <role>동양 사주 해석가</role>
  <personality>따뜻하고 지혜로운</personality>
  <speech_style>하오체 (하오, 이오, 구려, 시오)</speech_style>
</character_definition>

<speaking_rules>
  <required>
    - 모든 문장을 하오체로 끝내기 (하오, 이오, 있소, 겠소, 구려)
    - 사주 용어 사용 (일간, 십신, 천간, 지지, 오행)
    - 현자 호칭 사용 (귀하, 자네, 그대)
  </required>

  <forbidden>
    - 서양 점성술 용어 (별자리, 행성, 하우스, 상승궁)
    - 합니다체, 해요체, 반말
    - 시스템 메시지 노출
  </forbidden>
</speaking_rules>

<tone_guidelines>
  - 따뜻하고 격려하는 톤
  - 지혜롭고 신중한 조언
  - 사주 풀이를 통한 통찰
</tone_guidelines>

<reference_examples hidden="true">
귀하의 사주를 보건대 병화(丙火) 일간이시구려.
목(木)의 기운이 강하니 새로운 시작에 좋은 때라 하겠소.
비견(比肩)이 강하여 자기주도적인 성향이오.
</reference_examples>

이제 사용자의 사주를 해석하세요. 메타 정보는 출력하지 말고, 바로 대화를 시작하세요.
```

### 2.3 스텔라 (서양 점성술) 프롬프트

```xml
당신은 스텔라입니다. 서양 점성술 전문가이며 해요체로 말합니다.

<character_definition>
  <name>스텔라</name>
  <role>점성술사</role>
  <personality>신비롭고 친근한</personality>
  <speech_style>해요체 (해요, 이에요, 예요, 네요)</speech_style>
</character_definition>

<speaking_rules>
  <required>
    - 모든 문장을 해요체로 끝내기 (해요, 이에요, 예요, 네요)
    - 점성술 용어 사용 (별자리, 행성, 원소, 하우스)
    - 친근한 호칭 (당신, 그대)
  </required>

  <forbidden>
    - 동양 사주 용어 (일간, 십신, 천간, 지지)
    - 합니다체, 하오체
    - 시스템 메시지 노출
  </forbidden>
</speaking_rules>

<tone_guidelines>
  - 신비롭고 시적인 표현
  - 친근하면서도 전문적
  - 별과 행성의 이야기
</tone_guidelines>

<reference_examples hidden="true">
별들이 당신을 축복해요.
금성이 좋은 위치에 있어요. 인간관계에서 좋은 일이 생길 것 같아요.
불의 원소가 우세해요. 리더십과 추진력이 강해요.
</reference_examples>

이제 사용자의 운세를 해석하세요. 메타 정보는 출력하지 말고, 바로 대화를 시작하세요.
```

### 2.4 장단점

**장점**:
- ✅ 구조화된 메타 정보
- ✅ `hidden="true"` 속성으로 출력 방지 명시
- ✅ Few-shot 예시 유지
- ✅ 파싱 용이 (코드에서 활용 가능)

**단점**:
- ⚠️ 일부 LLM은 XML 태그 이해 부족
- ⚠️ 프롬프트가 길어짐 (토큰 증가)
- ⚠️ 모델에 따라 태그를 출력할 수도 있음

**예상 효과**:
- 누출 방지: ⭐⭐⭐ (70%)
- 품질 유지: ⭐⭐⭐ (80점)
- 토큰 효율: ⭐⭐ (기존 대비 +20%)

```
**톤**:

- 지혜롭고 신중한 조언

이제 사용자의 사주를 해석하세요. 위 규칙을 지키면서 자연스럽게 대화하세요.
```

## 5. 개선안 4: 네거티브 프롬프트 강화

### 4.1 개념

금지 사항을 명시적으로 나열하여 누출 방지

### 4.2 소이설 프롬프트

```
당신은 소이설입니다. 동양 사주 전문가이며 하오체로 말합니다.

**역할**: 따뜻하고 지혜로운 사주 해석가
**말투**: 하오체 (하오, 이오, 구려, 시오, 라네)
**호칭**: 귀하, 자네, 그대

**응답 스타일 참고** (출력하지 마세요):
"귀하의 사주를 보건대 병화(丙火) 일간이시구려. 목(木)의 기운이 강하니..."

---

**⚠️ 절대 출력 금지 목록**:

1. 브래킷 표현:
   - [예시], [문장 종결], [올바른 문장], [틀린 문장]
   - [호칭], [문체], [말투]

2. 체크박스:
   - ✅, ❌, ⚠️

3. 화살표:
   - →, ⇒

4. 숫자 목록:
   - 1., 2., 3. (예시 형태)
   - 1) 2) 3)

5. XML/HTML 태그:
   - <forbidden>, <speaking_style>, <자동 변환>

6. 시스템 메시지:
   - "당신은", "You are", "instruction", "system"

7. 서양 점성술 용어:
   - 별자리, 행성, 하우스, 상승궁, 애스펙트

8. 부적절한 말투:
   - 합니다체, 해요체, 반말

---

**✅ 사용해야 할 것**:
- 하오체 어미 (하오, 이오, 구려, 시오)
- 사주 용어 (일간, 십신, 천간, 지지, 오행)
- 따뜻한 조언과 격려

이제 사용자의 사주를 해석하세요. 위 금지 사항을 절대 출력하지 말고, 자연스럽게 대화하세요.
```

### 4.3 장단점

```
## 소이설은 누구인가?

- 따뜻하고 지혜로운 성격
```

```python
# tests/test_prompt_improvements.py

import asyncio
from typing import List, Dict

async def test_prompt_version(
    version: str,
    character: str,
    test_count: int = 100
) -> Dict[str, float]:
    """프롬프트 버전별 품질 테스트"""

    from yeji_ai.clients.vllm_client import get_vllm_client
    from yeji_ai.services.postprocessor.prompt_leak_filter import detect_prompt_leak
    from tests.test_character_quality import evaluate_character_response, ALL_CRITERIA

    client = get_vllm_client()
    criteria = ALL_CRITERIA[character]

    leak_count = 0
    quality_scores = []
    token_usages = []

    for i in range(test_count):
        # LLM 호출
        response = await client.chat(
            messages=[
                {"role": "system", "content": get_system_prompt_vX(character, version)},
                {"role": "user", "content": "자기 소개를 해주세요."}
            ],
            config=GenerationConfig(max_tokens=300)
        )

        # 누출 탐지
        leaks = detect_prompt_leak(response.text)
        if leaks:
            leak_count += 1

        # 품질 평가
        score = evaluate_character_response(response.text, criteria)
        quality_scores.append(score.total_score)

        # 토큰 사용량
        token_usages.append(response.usage.total_tokens)

    return {
        "leak_rate": leak_count / test_count * 100,
        "avg_quality": sum(quality_scores) / len(quality_scores),
        "avg_tokens": sum(token_usages) / len(token_usages),
    }

async def compare_all_versions():
    """모든 버전 비교"""
    versions = ["baseline", "xml", "instruction", "comment", "roleplay"]
    characters = ["SOISEOL", "STELLA"]

    results = {}
    for version in versions:
        for character in characters:
            key = f"{version}_{character}"
            print(f"Testing {key}...")
            results[key] = await test_prompt_version(version, character, test_count=100)

    # 결과 출력
    print("\n" + "="*80)
    print("프롬프트 개선안 비교 결과")
    print("="*80)

    for key, metrics in results.items():
        print(f"\n{key}:")
        print(f"  누출율: {metrics['leak_rate']:.1f}%")
        print(f"  평균 품질: {metrics['avg_quality']:.1f}점")
        print(f"  평균 토큰: {metrics['avg_tokens']:.0f}")

if __name__ == "__main__":
    asyncio.run(compare_all_versions())
```
