# 프롬프트 품질 비교: ai/main vs ai/develop

**분석 일시**: 2026-02-01  
**분석 대상**: yeji-ai-server 프롬프트 시스템  
**비교 브랜치**: ai/main → ai/develop

---

## 요약

`ai/develop` 브랜치는 `ai/main` 대비 **대규모 프롬프트 품질 개선**을 포함합니다.

**주요 변경사항**:
- 🆕 **티키타카 대화 시스템 완전 신규 구현** (`tikitaka_prompts.py` 추가)
- 🆕 **카테고리별 그리팅 시스템 추가** (`category_greetings.py` 추가)
- ✨ **페르소나 프롬프트 대폭 강화** (소이설, 스텔라 등 6개 캐릭터)
- 🛡️ **프롬프트 누출 방지 강화** (XML 태그, 메타 텍스트 출력 차단)
- 📝 **응답 품질 제어 개선** (문장 수, 반복 방지, 맥락 유지)

**총 변경량**: 9개 파일, +748줄, -168줄

---

## 1. 주요 변경 사항 요약

| 항목 | ai/main | ai/develop | 개선점 |
|------|---------|------------|--------|
| **티키타카 시스템** | ❌ 없음 | ✅ 완전 구현 | 대결/합의/첫인사/종료 모드 지원 |
| **카테고리 그리팅** | ❌ 없음 | ✅ 6개 카테고리 | 운세 유형별 맞춤 인사 |
| **말투 규칙** | 간단 (2-4문장) | 상세 (5-7문장, 100~600자) | 풍부한 표현, 반복 방지 |
| **프롬프트 누출 방지** | 기본 | 강화 (`<internal_only>` 태그) | XML 태그, 예시 출력 완전 차단 |
| **페르소나 깊이** | 기본 성격 설명 | 상세 (상호작용 패턴 포함) | 반박/동조/떠보기 패턴 정의 |
| **출력 검증** | 기본 | 엄격 (숫자 합계, 필드 검증) | percent 합계=100 강제 |
| **언어 순수성** | 기본 | 강화 (`<language_purity>`) | 깨진 문자, 외래어 혼용 차단 |

## 2. 프롬프트 구조 변화

### 2.1 티키타카 대화 시스템 (신규)

**ai/main**: 존재하지 않음

**ai/develop**: 완전 신규 구현 (`tikitaka_prompts.py`)

- 캐릭터 간 상호작용 패턴 정의 (반박, 동조, 떠보기)
- 발화 순서 제어 (2~4개 라인, 순서 엄격 준수)
- 감정 표현 강화 (emotion_code, emotion_intensity)
- 상대 캐릭터 이름 직접 언급 강제

### 2.2 페르소나 프롬프트 강화

#### 소이설 (SOISEOL) 비교

| 요소 | ai/main | ai/develop |
|------|---------|------------|
| **시스템 프롬프트 길이** | ~30줄 | ~80줄 (2.7배 증가) |
| **말투 규칙** | 간단 변환 예시 | 상세 변환 테이블 + 금지 어미 명시 |
| **금지 표현** | 1개 섹션 | 3단계 (어미/단어/이모지) |
| **상호작용 패턴** | ❌ 없음 | ✅ 추가 (반박/동조/떠보기 예시) |
| **프롬프트 누출 방지** | 기본 | 강화 (`<internal_only>`, XML 태그 차단) |
| **응답 길이** | 2-4문장 | 5-7문장 (100~600자) |

## 3. 품질 제어 메커니즘

### 3.1 프롬프트 누출 방지 강화

**ai/develop 개선점**:
- XML 태그 이름 명시적 금지 (`<persona>`, `<speaking_rule>` 등)
- 메타 텍스트 유형 구체화 ("필수 어미", "금지 어미" 등)
- `<internal_only>` 태그로 참조용 섹션 격리
- 반복 방지 지시 추가

### 3.2 응답 품질 제어

**ai/main**: 2-4문장으로 간결하게 답변

**ai/develop 개선**:
- 5-7문장으로 풍부하게 답변
- 글자 수 범위 추가 (100~600자)
- 상대 캐릭터 반응 패턴 명시
- 반복 방지 지시 추가
- 구체성 강조 (사주/별자리 요소 언급)

### 3.3 언어 순수성 강화

**ai/develop 신규 추가**:
```
<language_purity>
반드시 한국어로만 응답하시오.
허용: 한글, 한자 어원, 고유명사
금지: 깨진 문자, 영어/일본어 혼용, 의미 없는 특수문자
</language_purity>
```

## 4. 페르소나 개선

### 4.1 상호작용 패턴 추가

**ai/main**: 없음

**ai/develop**: 모든 캐릭터에 상호작용 패턴 추가

- 반박: 5개 패턴 (상대 의견 직접 반박)
- 동조: 3개 패턴 (합의 시 놀라움 표현)
- 떠보기: 2개 패턴 (상대 의견 유도)

### 4.2 캐릭터별 고유성 강화

| 캐릭터 | ai/main | ai/develop 추가 요소 |
|--------|---------|---------------------|
| **소이설** | 기본 말투 | 사자성어 5개, 기(氣) 표현 5개, 오행 표현 5개 |
| **스텔라** | 기본 말투 | 별/우주 표현 5개, 행성 표현 5개, 희망적 표현 4개, 허당끼 표현 3개 |
| **청운** | - | 시적 표현, 자연 비유 |
| **화린** | - | 비즈니스 비유, 나른함 |
| **카일** | - | 도박/게임 용어 |
| **엘라리아** | - | 별/빛 비유, 우아함 |

## 7. 리스크 및 주의사항

### 7.1 토큰 사용량 증가

- **ai/main**: 평균 ~300 토큰/프롬프트
- **ai/develop**: 평균 ~600 토큰/프롬프트 (2배 증가)

**완화 방안**: 불필요한 `<internal_only>` 섹션 최소화, 중복 설명 제거

### 7.2 복잡도 증가

- 프롬프트 파일 9개 → 유지보수 부담 증가
- 캐릭터 추가 시 일관성 유지 어려움

**완화 방안**: 공통 프롬프트 모듈화, 캐릭터 페르소나 템플릿 표준화

### 7.3 프롬프트 엔지니어링 의존성

- 말투 규칙 복잡도 증가 → LLM 준수율 저하 가능
- Qwen3 이외 모델 호환성 미검증

**완화 방안**: 후처리기(`postprocessor/`) 병행 활용, 말투 검증 로직 강화
